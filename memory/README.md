# 精细管理者 (Lean Manager) - 认知记忆引擎库

本目录 (`lean-manager/memory/`) 充当 AI Agent 执行精细管理任务时的**长期记忆皮层**。 
当遇到新类型的业务对象、解决生僻报错或完成极复杂工作流操作时，Agent 应主动把“新发现”、“新规律”记录在这里，并在后续类似任务中优先翻看此处经验。

## 记忆库结构概览

1. **`domain-patterns.md` (或按具体项目命名体系)**：
   - 用途：存储特定领域的已知固化内容：如常去爬取或审批的某个 Base 的 `app_token`、某重要群聊的 `chat_id` 或者某种特定审批任务的表单 JSON 格式等。
   - 这能避免在下次对话中向用户反复索要长难串行的飞书 Token。

2. **`troubleshooting.md` (报错反思库)**：
   - 用途：收录从 `Forbidden`、`429 Rate Limit` 到 `Field mismatch` 各种飞书底层报错或执行卡点时的终局解决配方。
   - 是提升任务排错速度的“绝杀大辞典”。

3. **`team-context.md` (组织与协同元数据)**：
   - 用途：记录人与人在特定系统内的偏好。
   - **身份记忆**：记录特定部门或任务最适合的 Auth 身份（如：财务审计必须用 `user`，周报广播建议用 `bot`）。

## Agent 定向写入范例：

如果 AI Agent 刚成功排查了一个“因为时间戳单位问题导致 Base 无法写入”的错误，它应该自行触发 `write_to_file` 或 `append`（视需求）到 `troubleshooting.md` 中，内容形如：
```md
*(2026-04-07)* ⚠️ **Base 表格日期类型写入失败的陷阱**  
**症状**：报错码 1254064 (Field Value Invalid)。
**原因**：因为提供的是秒级别 (`1712496000`) 或者是 ISO Format 字符串。
**正解解法**：必须使用且仅使用 13 位毫秒级时间戳 (如：`1712496000000`)。务必在通过 API 读取现有日期项时，将其处理成此格式再回写。
```

## Agent 定向读取准则：

- 在接收到一项包含大量不确定变量的长链路任务（如：“汇总本周产品研发部所有的上线需求单，找出瓶颈”）时，不要上来就死敲 API。请**先**来这个目录下翻阅有无可复用的 `domain-patterns`。
